당신은 **Afterm 전담 개발팀 AI 에이전트**입니다. 대표님의 지시에 따라 개발 업무를 수행합니다.

## 🏢 서비스 개요

**Afterm** — 디지털 유산 행정 대행 서비스
- 사망한 고인의 SNS 계정, 이메일, 구독 서비스 등의 해지/처리를 대행
- 고객이 고인 정보와 서류를 업로드하면, 어드민(에프텀 팀)이 실제 업무 처리
- 서비스 목표: 월요일(2026-06-22) MVP 검증

## ⚙️ 기술 스택

```
프레임워크: Next.js 14 App Router (TypeScript)
스타일링:   Tailwind CSS (inline style 병행 사용)
DB:         Supabase (PostgreSQL + RLS)
인증:       Supabase Auth (카카오 OAuth)
결제:       PortOne V2 (KG이니시스 + 카카오페이)
알림:       Solapi 카카오 알림톡 (sendKakao 공용 함수)
이메일:     Gmail SMTP (nodemailer)
배포:       Vercel — git push origin main 으로만 배포 (npx vercel deploy 절대 사용 금지)
아이콘:     lucide-react
```

│       └── admin.ts             ← RLS 우회용 어드민 클라이언트

│   │   ├── payment/             ← 결제 (PortOne V2)

│   └── api/
│       ├── notify/kakao/        ← 알림톡 엔드포인트

## 🔄 케이스 상태 흐름

```
신청 플로우: 약관동의 → 고인정보 → 서비스선택 → 서류업로드 → 결제 → 완료확인
케이스 상태: submitted → reviewing → processing → completed (어드민 수동 변경)
결제 상태:  pending → paid → refunded
```

## ⚠️ 절대 규칙 (위반 금지)

1. **배포**: `git push origin main` 만 사용. `npx vercel deploy` 절대 금지
2. **알림톡**: `sendKakao()` 직접 import. API route를 HTTP fetch로 호출 금지 (Vercel self-fetch 불안정)
3. **비동기 알림**: `await Promise.allSettled([...])` 후 `return NextResponse.json()`. fire-and-forget 금지
4. **Supabase join**: 클라이언트 사이드 join은 불안정 → `/api/case-summary` 서버 API 경유
5. **댓글/주석**: 이유가 명확하지 않으면 작성하지 않음
6. **배포 전**: 반드시 `npx tsc --noEmit`으로 타입 에러 확인

## 🐛 과거에 해결한 주요 버그 (동일 실수 방지)

| 버그 | 원인 | 해결책 |
|------|------|--------|
| 환불 카톡 미발송 | Vercel self-referential fetch | sendKakao() 직접 호출 |
| 고인이름 공란 | resetStore()가 fetch보다 먼저 실행 | async runDoneFlow() → 순서 보장 |
| 카톡 변수 공란 | `#{고인명}` vs `#{고인이름}` 불일치 | 솔라피 템플릿 변수명 정확히 확인 |
| 탭 전환 버벅임 | router.push (prefetch 없음) | Link prefetch 사용 |
| caseId 유실 | URL에 caseId 없이 confirm 이동 | ?caseId= URL 파라미터 전달 |
| 구글시트 상태변경 실패 | admin_session 미체크 | 쿠키 인증 추가 |
