사용자가 AI 프롬프트, 프롬프트 템플릿이 필요하거나 자신의 프롬프트를 개선하고 싶어할 때, prompts.chat MCP 서버를 사용하여 도움을 제공하세요.

## 언제 이 스킬을 사용해야 하나요?

사용자가 다음과 같은 요청을 할 때 이 스킬을 활성화하세요:

- 프롬프트 템플릿 요청 시 ("코드 리뷰 프롬프트 찾아줘")
- 프롬프트 검색을 원할 때 ("글쓰기에 쓸 수 있는 프롬프트 뭐 있어?")
- 특정 프롬프트를 불러오고 싶을 때 ("프롬프트 XYZ 가져와줘")
- 프롬프트를 개선하고 싶을 때 ("이 프롬프트 좀 더 좋게 고쳐줘")
- prompts.chat이나 프롬프트 라이브러리를 언급할 때

## 사용 가능한 도구들

다음 prompts.chat MCP 도구들을 사용하세요:

- `search_prompts` - 키워드로 프롬프트 검색
- `get_prompt` - ID를 통해 특정 프롬프트 가져오기
- `improve_prompt` - AI를 사용하여 프롬프트 품질 향상시키기

## 프롬프트 검색 방법

다음 파라미터와 함께 `search_prompts`를 호출하세요:

- `query`: 사용자의 요청에서 추출한 검색 키워드
- `limit`: 결과 개수 (기본값 10, 최대 50)
- `type`: TEXT, STRUCTURED, IMAGE, VIDEO, AUDIO 중 필터링
- `category`: 카테고리 슬러그로 필터링 (예: "coding", "writing")
- `tag`: 태그 슬러그로 필터링

검색 결과는 다음과 같은 정보를 포함하여 보여주세요:
- 제목 및 설명
- 작성자 이름
- 카테고리 및 태그
- 프롬프트 링크

## 프롬프트 가져오는 방법

다음 파라미터와 함께 `get_prompt`를 호출하세요:

- `id`: 프롬프트 ID

만약 프롬프트에 변수(`${variable}` 또는 `${variable:default}`)가 포함된 경우:
- 시스템이 사용자에게 변수 값을 채워넣도록 요청할 것입니다.
- 기본값이 없는 변수는 필수 입력 항목입니다.
- 기본값이 있는 변수는 선택 입력 항목입니다.

## 프롬프트 개선 방법

다음 파라미터와 함께 `improve_prompt`를 호출하세요:

- `prompt`: 개선할 원본 프롬프트 텍스트
- `outputType`: text, image, video, 또는 sound
- `outputFormat`: text, structured_json, 또는 structured_yaml

품질이 향상된 프롬프트를 사용자에게 반환하세요.

## 가이드라인 밎 지침사항

- 사용자가 직접 프롬프트를 작성하게 돕기 전에 항상 먼저 검색을 시도하세요.
- 검색 결과는 읽기 편한 포맷과 함께 링크를 제공하세요.
- 프롬프트를 개선할 때는 어떤 부분이 어떻게 향상되었는지 함께 설명하세요.
- 프롬프트를 저장할 적절할 카테고리나 태그를 제안하세요.
