1. 로그인 및 인증 (Login & Auth)
가장 시급한 문제입니다. "로그인은 되는데 정보가 제대로 안 넘어오는 현상"을 잡아야 합니다.
* [ ] Supabase Authentication 설정 확인
   * 대시보드 > Authentication > Providers에서 카카오/구글 Client ID와 Secret이 정확한지.
   * Redirect URL이 https://afterm.co.kr/auth/callback으로 등록되어 있는지.
* [ ] 미들웨어 세션 처리 (middleware.ts)
   * supabase.auth.getUser()가 매 페이지 요청마다 세션을 갱신하고 있는지 확인.
* [ ] 프로필 생성 트리거 (핵심)
   * 회원가입(auth.users 추가) 시 public.profiles에도 자동으로 데이터가 들어가는지 확인.
   * (이게 안 되면 모든 기능에서 유저가 '익명'이나 'Null'로 뜹니다. 아까 드린 fix_auth_trigger.sql이 이걸 해결합니다.)
2. 메시지 전달 프로세스 (Dead Man's Switch)
"생존 확인 메일이 안 가거나, 시간이 지나도 메시지가 안 풀리는 현상"을 잡아야 합니다.
* [ ] Cron Job 설정 (Vercel)
   * vercel.json에 crons 설정이 되어있고, Vercel 대시보드에서 Cron Job이 실제로 돌고 있는지.
* [ ] DB 상태값 (messages 테이블)
   * verification_status 컬럼이 report_received -> verifying_1 -> unlocked 순서로 잘 바뀌는지.
* [ ] 이메일 발송 (nodemailer)
   * 환경변수(GMAIL_USER, GMAIL_APP_PASSWORD)가 Vercel에 제대로 등록되어 있는지.
3. 온라인 추모관 (Online Memorial)
"댓글/좋아요는 써지는데 알림이 안 뜨거나 에러가 나는 현상"을 잡아야 합니다.
* [ ] DB 스키마 및 관계형성 (FK)
   * memorial_comments 테이블의 user_id가 auth.users를 제대로 참조하고 있는지. (이게 끊겨있으면 에러 발생)
* [ ] RLS (보안 정책)
   * public 권한으로 profiles 테이블을 읽을 수 있는지. (안 되면 프로필 사진 엑박)
   * memorial_likes 테이블에 insert 권한이 있는지. (안 되면 406 에러)
* [ ] 활동 로그 트리거 (activity_logs)
        •댓글/좋아요가 달릴 때 activity_logs 테이블에 자동으로 로그가 쌓이는지. (안 쌓이면 알림 0개)
