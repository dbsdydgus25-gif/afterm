프론트엔드 디버깅을 위한 상세 체크리스트를 준비했습니다.
프로젝트 분석 결과:
* Next.js 16 + React 19 (최신 버전): 호환성 이슈가 발생할 수 있는 환경입니다.
* Supabase Auth + Middleware: 인증 토큰 관리 및 리다이렉트 로직이 중요합니다.
* Framer Motion: 애니메이션이 많아 성능 및 레이아웃 이동(CLS) 확인이 필요합니다.
아래는 우선순위별 상세 디버깅 리스트입니다.
1. 핵심 기능 (Core Logic)
* 로그인/회원가입 흐름 (Auth Flow)
   * middleware.ts: 로그인 상태에서 /login 접근 시 리다이렉트가 잘 되는지?
   * src/app/auth: 소셜 로그인(Kakao 등) 후 세션(sb-access-token)이 쿠키에 정상적으로 저장되는지?
   * 새로고침 시 로그인 풀림 현상이 없는지? (Supabase onAuthStateChange 확인)
* 결제 및 플랜 (Payment & Plans)
   * src/app/plans: 'Basic' vs 'Pro' 플랜 전환 시 상태 업데이트(useMemoryStore)가 즉시 반영되는지?
   * 결제 성공(payment/success) 후 DB에 플랜 정보가 올바르게 업데이트되는지?
* 데이터 생성 (Create/Vault)
   * 유언장/메시지 작성 중 뒤로가기 눌렀을 때 임시 저장(Draft) 기능이 동작하는지? 또는 경고창이 뜨는지?
   * 이미지 업로드 시 파일 크기 제한(Supabase Storage Policy)에 걸리지 않는지?
2. UI/UX 및 레이아웃 (Visual Debugging)
* 반응형 디자인 (Mobile Response)
   * src/app/page.tsx: 모바일(375px~), 태블릿(768px~)에서 Hero 섹션 글자가 겹치거나 잘리지 않는지?
   * BottomNav가 모바일 키보드가 올라왔을 때 화면을 가리지 않는지? (iOS Safari 안전 영역 확인)
* 애니메이션 (Framer Motion)
   * AnimatePresence: 스플래시 스크린(Splash Screen)이 사라질 때 깜빡임(Flicker) 없이 자연스러운지?
   * 스크롤 시 등장하는 요소들(viewport={{ once: true }})이 새로고침 후에도 정상 작동하는지?
* 글로벌 스타일 (Global CSS)
   * Tailwind v4 적용으로 인한 스타일 깨짐 확인 (특히 globals.css의 커스텀 유틸리티 클래스들).
   * iOS에서 폰트 렌더링(Line-height)이 안드로이드/PC와 다르게 보이는지?
3. 성능 및 네트워크 (Performance & Network)
* 브라우저 콘솔 (Console Errors)
   * Hydration Error (Text content does not match...): 서버/클라이언트 시간 차이, 난수 생성 등으로 인한 에러 확인.
   * 유효하지 않은 DOM 중첩 (예: <p> 안에 <div> 넣기) 경고 확인.
* 네트워크 탭 (Network Tab)
   * API 호출 중 401/403 (권한 없음) 에러가 숨겨져서 발생하고 있지 않은지?
   * 이미지 리소스(opengraph-image.png, 프로필 이미지)가 404 없이 로딩되는지?
4. SEO 및 메타데이터
* Open Graph (카톡/슬랙 공유 미리보기)
   * opengraph-image.png가 최신 버전(로고 수정됨)으로 캐시 없이 반영되었는지? (카카오톡 개발자 도구로 캐시 초기화 필요)
   * 각 페이지(page.tsx, plans/page.tsx)마다 metadata title과 description이 고유하게 설정되어 있는지?
